## 既知の制約・課題

### 1. Drive 画像表示の不安定さ

Drive の `/file/d/XXX/view` URL を `lh3.googleusercontent.com/d/XXX=w400` に変換しているが、Google の仕様変更や CORS で表示できないケースがある可能性。実機で問題があれば代替手段を検討。

### 2. 田んぼでの電波状況は未確認

実機テスト前。電波が弱い時の挙動(写真アップロード失敗・タイムアウト)はまだ対応していない。Step 2 でオフライン下書き機能を検討。

### 3. Service Worker のキャッシュ戦略

現状:静的ファイルのみキャッシュ。GAS / Drive 通信はキャッシュしない。
オフライン時にホーム画面を開いても、データは取れない。

### 4. Push 通知なし

設計原則として通知は使わない方針。「ホーム画面を開いた時に最新情報を見る」が前提。

### 5. 認証なし

URL を知っていれば誰でも書き込み可能。8人の信頼関係を前提とする設計。
