# 田んぼ帳(仮称) 要件定義書 v0.5

> **v0.4.1 からの主な変更点**
> - **8人の実名を反映**(ゆうた・さえ・ながはし・みさき・カトケン・ことり・しゅんき・こうすけ)
> - **マスター当番表を確定**(曜日担当が決定)
> - **「1日2回見回り」(朝・夕) を要件に反映**
> - **見回り記録に「やることリスト」をリファレンス表示として追加**
> - **LINE共有テキスト生成機能を新設**(記録後にコピーボタン)
> - **「今週の当番」編集機能を追加**(代行依頼とは別、臨機応変な書き換え用)
> - **見回り記録メモにプレースホルダ例示**(4項目を誘導、構造化はしない)
> - **固有名詞・道具を要件に反映**(掛矢、水口、落し口、健さん)

---

## 1. プロジェクト概要

### 1.1 プロジェクト名(仮称)

**田んぼ帳 (tambo-chō)**

- 「アプリ」「システム」ではなく「**帳面**」と位置づける
- 機能を盛り込むのではなく、書き留めていく場所
- 正式名称は8人グループとの議論で確定

### 1.2 背景
- 上野尻集落の田んぼ1枚(1反未満)を、農家ではない8人がフラットな共同体として共同で耕作・管理している。
- 水源は二系統:
  - **専用ライン**: 自分たちの小川 → ホース → 田んぼ(常時流入)
  - **補助ライン**: 大本の堤(共用) → じょうご → 共用水路 → 三つ又分岐 → 田んぼ(小川が細った時のみ)
- 共用設備の操作には、堤を共有する他の田んぼ持ち(健さん 等)との調整が必要。
- 既存サービス(paditch、水田ファーモ等)は個人/法人による経営体としての省力化が前提で、対等な共同体メンバーによる共同管理を支援するものは事実上存在しない。
- 評価軸は「効率化」ではなく「関わりの質」。設計の最上位原則は **シンプルさ**。

### 1.3 目的
8人グループの「**当番調整**」「**日々の記録**」「**共用設備の操作ログ**」をWebアプリでシンプルに支援し、**LINE での共有を補助**する。

### 1.4 設計の最上位原則:シンプルさ
- 認証なし(オープン)
- 通知なし(プル型)
- アプリは「記録の置き場」、LINE は「会話の場」として役割分担
- 機能を増やしたくなったら一度立ち止まる

### 1.5 非目的(やらないこと)
- IoTセンサーによる水位の自動計測
- 遠隔バルブ操作
- 営農管理(施肥、薬剤、収穫予測など)
- 個人ごとの貢献度スコア化・ランキング表示
- 集落全体の水利秩序のデジタル化
- LINE Bot連携(自動共有)
- 通知・リマインダー機能
- 厳格な認証・アクセス制御
- 警告色を用いた行動催促表示
- **タスク実施の強制チェック化**(チェックリストは「見るだけ」)

## 2. メンバーと運用ルール

### 2.1 メンバー(8人)

ゆうた、さえ、ながはし、みさき、カトケン、ことり、しゅんき、こうすけ

### 2.2 マスター当番表

| 曜日 | 担当 |
|------|------|
| 月曜 | ゆうた・さえ |
| 火曜 | ながはし・みさき |
| 水曜 | カトケン・ことり |
| 木曜 | しゅんき・こうすけ |
| 金曜 | ながはし・みさき |
| 土曜 | ゆうた・さえ |
| 日曜 | しゅんき・こうすけ |

**メモ**: カトケン・ことりは現時点で週1回。今後の調整によりバランスが変わる可能性あり。

### 2.3 見回りの実施タイミング(1日2回)

**朝の見回り(6:00 - 7:30 頃)**
タスク:
- 水量チェック(目安:苗半分)
- 足りなければじょうご開け・水入れ
- じょうごは必ず閉める(掛矢を使う)
- 水口も閉める(勝手に水が入ってこないように)
- 水口・落し口の決壊や崩れのチェック
- 崩れていたら泥で適宜修復
- 適宜、畔まわり・田んぼ内の草刈り

**夕の見回り(17:00 - 19:00 頃)**
タスク:
- 水量チェック
- 足りなければ水入れ、または次の当番に連絡
- ※水入れは基本日中。日暮れてからは水止めるが原則

### 2.4 報告・共有(原則:朝と夕それぞれ)

**共有先**: 全体LINEグループ

**報告内容**:
- 田んぼの状態
- やったこと
- 健さんや集落の人から言われたこと
- 懸念や課題

**アプリの役割**: 記録 + LINE共有テキストの生成支援(コピーボタン)。LINE 自動投稿は行わない。

### 2.5 設備・用具・関係者の用語集

| 用語 | 説明 |
|------|------|
| 掛矢(かけや) | じょうご開閉に使う木槌 |
| 水口(みなくち) | 田んぼへの水の入口 |
| 落し口(おとしぐち) | 田んぼからの水の出口 |
| じょうご | 大本の堤の栓。共用 |
| 三つ又 | 共用水路から各田んぼへの分岐点。共用 |
| 健さん | 集落の方(他の田んぼ持ち、調整相手) |
| 苗半分 | 標準の水位目安(稲の半丈) |

## 3. 機能要件

### 3.1 当番カレンダー(曜日担当制 + 臨機応変な編集)

**マスター当番表**(§2.2 参照)
- 月〜日、各曜日に2人ずつ割り当て(編集可能)
- 編集は専用画面から(誤操作防止のため、ホームに直接の編集UIは置かない)

**今週の当番(臨機応変な書き換え)**
- 「今週の当番」表は、マスター当番表を初期値として生成される
- メンバーは**自由に書き換え可能**(代行依頼フローを経ずに直接編集できる)
- 例:「今週金曜は ながはし→こうすけ」と直接置き換える
- 書き換え履歴は記録される(誰が誰に変えたか、いつ)

**代行依頼フロー**(残す)
- 「○月○日の自分の当番、誰か代わって」という意思表示が必要なケース用
- 「依頼」「受諾」のフロー
- ただし、軽い書き換えは§3.1 の「臨機応変な編集」で直接行える

**ホーム表示**
- 今日の当番(2人表示)
- 「今週の当番」を確認・編集できる導線

### 3.2 日々の見回り記録

**入力項目**
- 訪問者(自分の名前を選択、8人から)
- 訪問日時(デフォルトは現在)
- 水位写真(稲と定規を当てたもの)
- 水位評価(高 / 適 / 低)※「本日の水加減」表示用
- 小川(ホース)の水量状態 → 通常 / 弱い / ほぼなし
- やったこと(自由メモ、プレースホルダで誘導)
- 自由メモ(プレースホルダで4項目を例示)

**メモのプレースホルダ例**:
```
例:
・田んぼの状態 → 水位適、稲は元気
・やったこと → じょうご開けて水入れた
・健さん・集落から → 水路掃除が来週あるかも
・懸念 → 南東の畔が崩れそう
```

入力は完全自由。書きたい人は項目分けして書く、忙しい人は一言だけでもOK。

**朝/夕の区別**
- 別記録として扱わない(時刻で自然と区別される)
- 「朝の見回り」「夕の見回り」というラベルは付けない(投稿時刻で判断)

### 3.3 やることリスト(リファレンス表示)

見回り記録画面の上部に、**朝のタスク・夕のタスク** をリファレンスとして表示。
- チェックボックスではなく **読み物として表示**
- 入力フォームとは独立
- 訪問時刻(朝/夕)によって自動的に該当タスクを上に表示

### 3.4 共用設備の操作ログ

**入力項目**
- 操作者
- 操作対象: じょうご / 三つ又 / その他
- 操作内容:
  - じょうご: 開けた / 閉めた(掛矢使用)
  - 三つ又: 画像アップ(必須)
  - その他: 自由記述 + 画像(任意)
- 時刻(デフォルトは現在)
- 操作した理由(任意)
- 健さん・他の田んぼ持ちとの調整内容(自由メモ)

**じょうごの開け閉めの注意喚起**:
- 「開けた」を記録した時、「**必ず後で閉める!**」と画面に注意喚起を表示
- まだ閉めていない「開けた」記録があれば、ホーム画面に**さりげなく**表示する(警告ではなく、忘れ防止のリマインダー)

### 3.5 LINE共有テキスト生成(コピーボタン)

**位置づけ**: アプリへの記録後、LINE共有用テキストを自動生成。クリップボードへコピー → LINEグループに貼り付け。

**見回り記録のテンプレート例**:
```
🌾 田んぼ報告 ながはし
2026/5/16 7:15(朝の見回り)

【状態】水位:適 / 小川:通常
【やったこと】じょうご開けて水入れた
【健さん・集落から】明日水路掃除があるかも
【懸念】南東の畔が崩れそう
```

**共用設備操作のテンプレート例**:
```
🚰 共用設備操作 ながはし
2026/5/16 18:30

じょうごを開けた
理由:小川が細っていたため
調整:中田さんと話して10分だけ
※明朝、忘れずに閉めにいきます
```

**動作**:
- 記録完了画面に「LINE用テキストをコピー」ボタン
- ワンタップでクリップボードへ
- ユーザーはLINEを開いて貼り付け
- アプリは LINE には直接送信しない(独立性保持)

### 3.6 自由ノート(覚書)

- 日付に紐付かない長期的メモを置く場所
- 全員が編集可能
- ピン留め機能(「留め書き」)= 重要なノートを上に固定
- 例:「中干しは6月末から7月20日まで」「水路掃除◯日」「健さんの連絡先」

### 3.7 本日の水加減(目標と直近の並列表示)

**位置づけ**: ホーム画面の最上部に常時表示。

**左:目標**
- 今日の日付から自動的に時期を判定し、目標水位を表示
- 時期テーブル(初期値):
  - 5月: 苗半分
  - 6月: 浅め
  - 6月末〜7月20日: 中干し(ひび割れるぐらい)
  - **7月20日以降は要追記**
- SVGの簡易イラスト + テキスト + 補足

**右:直近**
- 直近の見回り記録から自動取得(時刻問わず最新の1件)
- 写真サムネ・訪問者名・経過時間(グレー、警告色なし)
- 主観評価タグ
- タップで詳細画面へ遷移

**じょうご開け閉め未完了表示**(追加)
- 「開けたまま閉めていない」じょうご操作があれば、ここに**さりげなく**表示
- 例:「ながはしが昨日18:30 にじょうごを開けたまま」
- 「閉めた」記録が出るまで表示

### 3.8 履歴ビュー(綴り)

- 時系列タイムライン
- フィルタ: すべて / 見回り / 共用設備
- 日付ごとにグルーピング(今日 → 昨日 → 一昨日 ...)
- 水位写真のサムネ表示

## 4. 非機能要件

| 項目 | 要件 |
|------|------|
| 規模 | 8ユーザー、田んぼ1枚、シーズン約150日、1日2回見回り想定 |
| デバイス | スマホ主、PCも可 |
| 通信 | 田んぼで電波が届く前提(**要現地確認**) |
| PWA | ホーム画面追加対応、オフライン時は最低限の閲覧可 |
| データ保存(構造) | Google Sheets |
| データ保存(画像) | LONGBRIDGE法人 Google Drive |
| 画像圧縮 | フロント側で長辺1920px・JPEG品質80% |
| 1枚あたり想定容量 | 200〜500KB |
| シーズン総容量見積もり | 約600MB〜1GB(1日2回 × 150日 × 平均2-3枚 = 約900〜1500枚) |
| 認証 | なし(URLを知っていれば誰でもアクセス可) |
| 運用コスト | 月額0円(GitHub Pages + GAS + Sheets はすべて無料) |
| バックアップ | Sheetsの自動バージョン履歴、Drive のファイル復元機能 |

### 4.1 オープン構成のリスクと割り切り
- URLが外部に漏れれば誰でも閲覧・編集可能
- 8人の信頼関係を前提にする
- 個人情報や機密情報は載せない方針で運用
- 万一の改ざんはSheetsのバージョン履歴から復元可

### 4.2 画像保存の運用方針

**保存先**: LONGBRIDGE法人 Google Drive

**フォルダ構造**:
```
LONGBRIDGE Drive
└── 田んぼ帳/
    ├── 2026年/
    │   ├── 見回り写真/
    │   │   ├── 20260516_ながはし_0715.jpg
    │   │   └── ...
    │   └── 共用設備写真/
    │       ├── 20260515_ながはし_じょうご_1830.jpg
    │       └── ...
    └── 2027年/
```

**ファイル命名規則**: `YYYYMMDD_名前_カテゴリ_HHMM.jpg`
- カテゴリ:見回り写真は省略、共用設備は「じょうご」「三つ又」等

**共有設定**: 「リンクを知っている全員が閲覧可」
- 編集権限は GAS のみ
- 8人グループは閲覧可能

**圧縮処理**: フロント側で JavaScript(Canvas API)で実施(1920px / JPEG 80%)

**位置づけの透明性**:
- 写真は LONGBRIDGE が管理する Drive に保存される旨を、8人グループに明示
- 初回URL共有時の説明文に含める、または覚書(notes)に固定で記載

## 5. 設計原則

### 5.1 シンプルさが最上位
- 機能を増やしたくなったら立ち止まる
- 認証・通知・自動連携などのレイヤーは避ける

### 5.2 プル型を徹底
- 通知で能動的に呼びかけない
- メンバーが自分のタイミングで見にくる前提
- 共有は人間が LINE で行う(アプリはコピーテキストを用意するのみ)

### 5.3 「記録できる」を最優先
- 当番調整・分析は二次的

### 5.4 警告色を使わない
- 「○日以上記録なし」を赤で警告するUIは作らない
- 経過時間表示はグレーのみ
- ただし「じょうご開けっぱなし」のような**事故防止リマインダー**はさりげなく表示する(これは警告ではなく安全のための情報)

### 5.5 タスクは「見るだけ」
- やることリストはチェックボックスにしない
- 業務管理ツール化を避ける
- 「自由意思で書く」体験を保つ

## 6. 技術スタック

```
ブラウザ(スマホ/PC)
   ├── PWAとしてホーム画面に追加
   ├── 画像はフロントで圧縮(1920px / JPEG80%)
   ├── 記録後にLINE共有テキストをクリップボードへ
   ↓
GitHub Pages (フロント)
   ↑↓ JSONP / fetch
GAS (Google Apps Script)
   ├──→ Google Sheets (構造データ)
   └──→ Google Drive [LONGBRIDGE法人] (画像)
              └── URLを Sheets に書き戻す
```

- **フロント**: GitHub Pages + React or Vanilla JS + Service Worker(PWA化)
- **バックエンド**: GAS(Webhook型のAPI)
- **データ**: Google Sheets
- **画像保存**: Google Drive(LONGBRIDGE法人)
- **クリップボードAPI**: ブラウザ標準の Clipboard API を使用
- 既存資産(taskmanager / tally)と同系統 → 流用効果大

### 8.2 ホーム画面(/)

- **本日の水加減**(目標+直近の並列)
- **じょうご未完了リマインダー**(該当時のみ、控えめに表示)
- **本日の当番**(2人、朱印風アイコン)
- **近ごろの見回り**(直近3件)
- **共用設備の動き**(直近2件)
- **覚書から**(留め書き1件)

### 8.3 見回り記録画面(/visit)

**上部:やることリスト(リファレンス表示)**
- 朝(6-7:30) / 夕(17-19)の現在時刻に応じて該当タスクをハイライト表示
- 読み物として、チェックボックスなし

**下部:入力フォーム**
- 名前(プルダウン)
- 水位写真(撮影/選択)
- 水位評価(高/適/低)
- 小川の状態(通常/弱い/ほぼなし)
- 自由メモ(プレースホルダで4項目を例示)
- 「記す」ボタン

**送信後**:
- 完了画面に「LINE用テキストをコピー」ボタン
- コピー → LINE への手動貼り付け

### 8.4 共用設備操作画面(/facility)

- 名前
- 操作対象(じょうご/三つ又/その他)
- 操作内容(対象により分岐、じょうごは「開けた/閉めた」)
- 「開けた」を選択した時、画面に**「必ず後で閉めましょう」** の注意喚起
- 写真(三つ又は必須)
- 理由
- 健さん・他の方との調整
- 「記す」ボタン

**送信後**:
- LINE用テキストコピーボタン

### 8.5 当番カレンダー画面(/duty)

**今週の当番セクション**
- 月〜日の7日間、各曜日2人ずつ
- 各人の名前をタップ → 別のメンバーに変更可能(直接編集)
- 「マスター通り」を朱で控えめに、「変更あり」は通常表示
- 変更履歴は別画面で見られる

**代行依頼セクション**
- 既存の依頼一覧(未受諾・受諾済)
- 新規依頼ボタン

**マスター当番表の編集**(別画面遷移、誤操作防止)
- 「マスター当番表を見直す」ボタン

### 8.6 覚書画面(/notes)

- 留め書き(ピン)セクション
- その他の覚書セクション
- FAB「書く」(右下、朱色)

### 8.7 履歴画面(/history)

- フィルタチップ(すべて/見回り/共用設備)
- 日付ごとのタイムライン
- 各記録:タグ + 時刻 + テキスト + サムネ
