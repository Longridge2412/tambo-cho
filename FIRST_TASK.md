# Cowork で最初にやること

このフォルダを Cowork で開いた時に、Claude が最初に取り組むタスクの指示書です。

---

## 0. まず読むもの

このファイルの後、以下の順で読んでください:

1. `HANDOFF.md` ── プロジェクト全体の概要、設計原則、現状
2. `docs/CURRENT_STATUS.md` ── 進捗の詳細と既知の課題
3. `docs/REQUIREMENTS_v0.5.md` ── 要件定義書(全体)

これらを読んで全体像を掴んでから、Yuki に「Step 1 のデプロイから始めますか?」と確認してください。

---

## 1. ユーザーへの最初の挨拶

挨拶は不要(Yuki は既に背景を知っている)。読み込み完了後、以下のように現状認識を簡潔に伝えると良い:

> 「読み込みました。Step 0 完了、Step 1 のコードが完成済みで、GAS への反映と GitHub デプロイがこれからの状態ですね。先に GAS の更新を進めますか?」

---

## 2. 当面の作業

### Phase A: Step 1 のデプロイ(優先)

1. **GAS のコード更新を Yuki にやってもらう**
   - `gas-files/Code.gs` の中身を、Yuki が GAS エディタにコピペして保存
   - 「デプロイを管理」→「新しいバージョン」でデプロイ
   - URL が変わらないことを確認

2. **Git 初期化 + GitHub にプッシュ**(Cowork が実行)
   ```bash
   git init
   git add -A
   git commit -m "Initial commit: 田んぼ帳 Step 1"
   git branch -M main
   gh repo create Longridge2412/tambo-cho --public --source=. --remote=origin --push
   ```
   - `gh` 未認証なら `gh auth login` を案内
   - `gh` 未インストールなら、ブラウザでリポジトリ作成 + `git remote add origin ...` の代替手順

3. **GitHub Pages を有効化**(Yuki に Web UI でやってもらう)
   - リポジトリ Settings → Pages → main / (root) → Save

4. **動作確認**
   - 公開URL `https://longridge2412.github.io/tambo-cho/` を開く
   - ホーム画面の各セクションが描画されるか
   - 見回り記録が動作するか
   - 写真アップロードが通るか
   - LINE 共有テキストのコピーが機能するか

5. **エラー対応**
   - ブラウザコンソール(F12 → Console)のエラーを Yuki にスクショや貼り付けで共有してもらう
   - 原因を特定して修正

### Phase B: スマホ動作確認

Yuki にスマホで PWA としてホーム画面追加してもらい、田んぼでの実機テストに備える。

### Phase C: フィードバック対応

実機テストで問題が見つかったら、その場で修正対応。

---

## 3. その後(Step 2 の予告)

Step 1 が安定したら、Step 2 に進む:

- 共用設備の操作画面(`#/facility`)
- 履歴画面(`#/history`)
- 当番カレンダー画面(`#/duty`)
- 覚書画面(`#/notes`)
- じょうご未完了リマインダー

ただし、Step 1 のフィードバックを十分に集めてから Step 2 に着手する。`HANDOFF.md` の「設計原則」に反する機能要望が出てきたら、それを指摘してから判断する。

---

## 4. 絶対にやってはいけないこと

- 警告色を使った督促表示の追加
- 個人ランキング・スコアリングの実装
- LINE Bot 連携の実装(共有テキスト生成までで止める)
- やることリストのチェックボックス化
- 8人の表示名を勝手に整形すること
- 既存の GAS デプロイ URL を変えてしまうこと(`src/config.js` を更新すれば変えられるが、それは最終手段)

---

## 5. 困った時の対応

- 仕様で迷ったら `docs/REQUIREMENTS_v0.5.md` を参照
- それでも判断つかなければ Yuki に確認
- コードに変更を加える時は、変更内容と理由を口頭で簡潔に説明してから進める

---

*このプロジェクトは「シンプルさが最上位」を絶対原則とする。機能追加の誘惑があったら、まず立ち止まる。*
